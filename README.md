# Apple Notes to Markdown Exporter

This project aims to export Apple Notes content into structured files (JSON or Markdown). We explored multiple approaches to extract note content with full semantic fidelity — including checkboxes, hyperlinks, formatting, and indentation — but encountered key limitations with every method except AppleScript.

## 🧭 Problem Summary

Apple Notes content is not directly accessible in a structured form. While the Notes app offers a `body()` method through scripting interfaces, it only exposes simplified HTML. Many important semantics — such as hyperlinks and checklists — are **not included** in the HTML or RTF generated by standard means.

---

## 🧪 Approaches We Explored

### ✅ 1. JXA → HTML
We accessed notes using JavaScript for Automation:

```js
note.body() // returns simplified HTML
```

**❌ Limitations:**
- `<li>` tags provide **no semantic distinction** between:
  - Bullet list
  - Numbered list
  - Checklist
- **Hyperlinks are omitted**
- **Checkbox state (checked/unchecked) is lost**

### ✅ 2. JXA → HTML → RTF (via `textutil`)
We converted HTML to RTF to capture richer formatting.

```sh
textutil -convert rtf -output note.rtf note.html
```

**❌ Still Missing:**
- Hyperlinks are flattened
- Checklists become generic bullets or glyphs
- No reliable marker for list types

### ✅ 3. Swift App (via `NSAttributedString`)
We built a native Swift CLI using `ScriptingBridge` to extract `NSAttributedString` representations of notes.

**Goal:** access attributes like `.link`, `presentationIntents`, and `.textLists`.

**❌ Findings:**
- Apple Notes **does not include** hyperlink targets (`.link` attributes are missing)
- `.presentationIntents` for checklists are **not set**, even for visual checkboxes
- We can detect formatting (bold, italic) and some list nesting via `.headIndent`, but **not checkbox state**

### ✅ 4. RTF Parsing (via `rtf-parser`)
We parsed Apple-generated `.rtf` files using low-level JavaScript libraries.

**❌ Findings:**
- Checkboxes (`☐`, `☑`) are present as Unicode characters
- But **list type**, indent level, and `\lsN` markers are not exposed in a structured form
- RTF semantics are inconsistently preserved across exports

---

## ✅ Final Approach: AppleScript Export

After evaluating all options, we settled on **AppleScript** as the simplest and most reliable method to extract note content in a structured format.

```applescript
tell application "Notes"
  set theNotes to notes of folder "..." of account "..."
  repeat with theNote in theNotes
    set noteTitle to name of theNote
    set noteContent to body of theNote
    ...
  end repeat
end tell
```

**✅ Pros:**
- Accesses all visible notes, folders, accounts
- Exports consistent content
- Can be saved as `.json` or `.md`

**❌ Cons:**
- No support for detecting hyperlinks (they are rendered visually, not as attributes)
- No way to detect checklist items or their status
- Tables and embedded content (images, sketches) are not accessible

---

## 📦 Export Format

Each exported note is saved as `.json`:

```json
{
  "title": "My Note",
  "content": "<div>Note content here</div>",
  "folder": "Work",
  "account": "iCloud",
  "created": "Wednesday, April 17, 2025 at 10:00:00 AM",
  "modified": "Wednesday, April 17, 2025 at 10:15:00 AM"
}
```

---

## 🔚 Conclusion

After extensive exploration, we concluded:

> **Only AppleScript provides a reliably automatable path to export Apple Notes.**  
> While richer semantics (like hyperlinks and checkboxes) exist visually, they are not exposed through any scripting interface or export format offered by Apple.

This solution prioritizes **stability and completeness** of raw content over perfect semantic fidelity.
