# Apple Notes Exporter

This project aims to export Apple Notes content. We explored multiple approaches to extract note content with full semantic fidelity — including checkboxes, hyperlinks, formatting, and indentation — but encountered key limitations with every method.

## 💹 Market Analysis

We tried many open source projects and commercial products and discovered that none of them manage to export the hyperlinks (NOTE: it is possible with Apple Notes Shortcuts) and checklists.

## 🧭 Problem Summary

Apple Notes content is not directly accessible in a structured form. While the Notes app offers a `body()` method through scripting interfaces, it only exposes simplified HTML. Many important semantics — such as hyperlinks and checklists — are **not included** in the HTML or RTF generated by standard means.

## 🧪 Approaches We Explored

### ✅ 1. SQlite

We started by trying to extract information directly from the SQlite Apple Notes uses.

Unfortunately, this is a not a reliable method since Apple Notes transitioned to iCloud as the data is not scrambled through multiple databases.

### ✅ 2. JXA → HTML

Then, we tried accessing notes using JavaScript for Automation (JXA):

```js
note.body() // returns simplified HTML
```

Even though it worked pretty smoothly, we encountered some limitations.

** ❌ Limitations:**
- `<li>` tags provide **no semantic distinction** between:
  - Bullet list
  - Numbered list
  - Checklist
- **Hyperlinks are omitted**
- **Checkbox state (checked/unchecked) is lost**

### ✅ 3. JXA → HTML → RTF (via `textutil`)

We tried converting HTML to RTF to capture richer formatting.

```sh
textutil -convert rtf -output note.rtf note.html
```

Strangely, converting a HTML that is missing metadata ends up providing more metadata. This is because textutil talks directly with macOS' Apple Notes to retrieve the original content.

Unfortunately there were still limitations.

** ❌ Still Missing:**
- Hyperlinks are flattened
- Checklists become generic bullets or glyphs
- No reliable marker for list types

### ✅ 4. Swift App (via `NSAttributedString`)

We then tried building a native Swift app using `ScriptingBridge` to extract `NSAttributedString` representations of notes.

This looked like the most complicated method but the one that would for sure give us everything we needed since it was directly communicating with Apple Notes in Apple's native language and framework.

The goal, more specifically, was to access metadata attributes like `.link`, `presentationIntents`, and `.textLists` that should hold the information that we nedded.

**❌ Findings:**
- Apple Notes **does not include** hyperlink targets (`.link` attributes are missing)
- `.presentationIntents` for checklists are **not set**, even for visual checkboxes
- We can detect formatting (bold, italic) and some list nesting via `.headIndent`, but **not checkbox state**

### ✅ 5. AppleScript Export

After evaluating all options, we settled on **AppleScript** as the simplest and most reliable method to extract note content in a structured format.

```applescript
tell application "Notes"
  set theNotes to notes of folder "..." of account "..."
  repeat with theNote in theNotes
    set noteTitle to name of theNote
    set noteContent to body of theNote
    ...
  end repeat
end tell
```

**✅ Pros:**
- Accesses all visible notes, folders, accounts
- Exports consistent content
- Generates simple HTML content

**❌ Cons:**
- No support for detecting hyperlinks
- No way to detect checklist items or their status

### 🔚 Conclusion

After extensive exploration, we concluded:

**Since there is no method for extracting metadata for hyperlinks and checklists, Apple Script is the most reliable way to automatable exporting Apple Notes.**  

## 👨🏻‍💻 Development

In order to set up the dev environment, follow the steps below (for macOS):

1. $> brew install npm
2. $> npm install
3. $> npm start

You can run the core scripts individually, specifically:

$> osascript scripts/list.applescript

## 📱 Application

The application follows a slightly unexpected procedure.

Indeed, because the Apple Script protocol is particularly slow, the number of calls to Apple Notes must be reduced as much as possible.

To give some context, listing 10,000 notes take about 20 hours.

The original flow went as follows:
1. Browse the hierarchy of notes (scripts/list.applescript)
2. Let the user select the notes to export
3. For each note to export:
  1. Export the note to HTML (scripts/export.applescript) in a temporary file
  2. Convert (if necessary) to the format the user picked

Unfortunately, the performance was catastrophic because going through the hiearchy already takes
the bulk of the time.

The exporting was therefore merged with the listing. The flow is therefore now:

1. Browse the hiearchy and export all notes in the app data directory
2. Let the user select the notes to export
3. For each note to export:
  1. Convert (if necessary) to the format the user picked
